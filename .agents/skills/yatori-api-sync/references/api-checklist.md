# API 对照清单

先看 `src/lib/api.ts`，再按下面顺序对官方文档。

## 1. 入口信息

- 基础地址是否变了
- 鉴权方式是否还是 `Authorization: Bearer <token>`
- SSE 地址和鉴权参数是否变了

## 2. 路径与方法

- 路径是否一致
- HTTP 方法是否一致
- 是否新增路径参数、查询参数或 body

## 3. 请求体

- 字段名是否一致
- 字段类型是否一致
- 可选 / 必填是否一致
- 数字枚举、字符串枚举是否变化

## 4. 响应体

- 顶层是否还是 `{ code, message, data }`
- `data` 是否为空、对象还是数组
- 时间字段是否还是字符串
- 新增字段是否要补进 TypeScript interface
- 废弃字段是否还在前端被使用

## 5. 业务状态

- 业务成功码是否仍是 `200` / `201`
- 是否有新的错误码要单独处理
- 401、404、部分成功这类边界是否影响 UI

## 6. 影响面

至少回查这些调用点：

- 登录
- 账号列表 / 当前账号
- 课程列表 / 课程详情
- 任务创建 / 启动 / 停止 / 进度
- 签到监控配置 / 启停 / 日志

## 7. 改动顺序

1. 改 `src/lib/api.ts`
2. 改调用方类型和字段读取
3. 跑 `npm run build`
4. 跑 `npm run lint`

## 8. 输出要求

在回复里明确列出：

- 哪些接口文档变了
- 哪些前端类型变了
- 哪些 UI 逻辑跟着变了
- 还剩哪些不确定项
