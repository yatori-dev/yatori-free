# 组件拆分清单

## 先判断是不是该拆

出现任一情况就该拆：

- 文件超过 300 行
- 一个组件同时管请求、缓存、视图和交互
- 同类 JSX 片段重复两次以上
- `useEffect` 超过两个且互相牵连
- 一个 state 改动会牵出多个不相关区域

## 拆分顺序

1. 静态区块
2. 可复用列表项
3. 工具函数
4. 异步请求逻辑
5. 持久化和流逻辑

## 每拆一步都要检查

- props 有没有变得更清楚
- 类型有没有跟着收口
- 父组件是否还保留真正的页面编排职责
- 目录名是否体现功能，而不是体现视觉

## 不要这样拆

- `utils.tsx` 这种垃圾桶文件
- 把一个巨型组件拆成五个巨型组件
- 为了抽而抽，搞出一堆单次使用 hook
