# UI 增量开发规则

## 先看什么

- `components.json`
- `src/index.css`
- 目标页面附近的现有组件

## 组件选择顺序

1. 现成 `src/components/ui/*`
2. 项目里已有业务组件模式
3. 再考虑新增 shadcn 组件

## 样式规则

- 优先 `bg-card`、`text-foreground`、`border-border` 这类语义 token
- 少写裸十六进制颜色
- 裸颜色只用于项目已经显式建立的品牌色场景

## 文案规则

- 文案只服务当前操作
- 不在界面里写“这个功能可以……”
- 不塞教程式解释

## 布局规则

- 保持当前应用的工具面板气质
- 少搞营销页式大块留白
- 移动端和桌面端都要看文字是否挤爆
